# WhisperShortcut Screenshots

This directory contains all the assets and scripts needed to generate App Store screenshots for WhisperShortcut.

## Directory Structure

```
screenshots/
├── assets/                  # Manually provided screenshots (UI assets)
│   ├── dropdown.png
│   ├── settings.png
│   └── settings-offline-models.png
├── output/                   # Generated App Store screenshots
│   ├── speech-to-text.png
│   ├── speech-to-prompt.png
│   ├── powered-by-gemini.png
│   └── powered-by-whisper.png
├── html/                    # HTML files for each screenshot
│   ├── speech-to-text.html
│   ├── speech-to-prompt.html
│   ├── powered-by-gemini.html
│   └── powered-by-whisper.html
├── js/                     # JavaScript capture scripts
│   └── capture-all.js
├── package.json            # Node.js dependencies
└── README.md              # This file
```

## Screenshot Descriptions

### 1. Speech-to-Text

- **File**: `html/speech-to-text.html`
- **Script**: `js/capture-all.js speech-to-text`
- **Output**: `output/speech-to-text.png`
- **Content**: Shows the core transcription functionality with step-by-step process

### 2. Speech-to-Prompt

- **File**: `html/speech-to-prompt.html`
- **Script**: `js/capture-all.js speech-to-prompt`
- **Output**: `output/speech-to-prompt.png`
- **Content**: Shows the AI prompt generation feature with voice input and response

### 3. Powered by Gemini

- **File**: `html/powered-by-gemini.html`
- **Script**: `js/capture-all.js powered-by-gemini`
- **Output**: `output/powered-by-gemini.png`
- **Content**: Shows the Gemini integration and advanced settings configuration

### 4. Powered by Whisper

- **File**: `html/powered-by-whisper.html`
- **Script**: `js/capture-all.js powered-by-whisper`
- **Output**: `output/powered-by-whisper.png`
- **Content**: Shows the offline Whisper models feature with privacy advantages and local large language model capabilities

## Asset Images

The following images are manually provided assets stored in the `assets/` directory that are used within the HTML files during screenshot generation:

- **assets/dropdown.png**: Screenshot of the WhisperShortcut menu bar dropdown interface
- **assets/settings.png**: Screenshot of the WhisperShortcut advanced settings window
- **assets/settings-offline-models.png**: Screenshot of the WhisperShortcut offline models settings tab

These assets are embedded in the HTML templates and appear as part of the generated screenshots. They should be updated manually when the UI changes.

## Usage

### Prerequisites

- Node.js installed
- Dependencies installed: `npm install`

### Generate All Screenshots

```bash
cd js
node capture-all.js
```

### Generate Specific Screenshot

```bash
cd js
node capture-all.js speech-to-text      # For speech-to-text
node capture-all.js speech-to-prompt   # For speech-to-prompt
node capture-all.js powered-by-gemini   # For Gemini settings
node capture-all.js powered-by-whisper  # For Whisper offline models
```

## Technical Details

- **Resolution**: All screenshots are generated at 1280x800 pixels
- **Format**: PNG with transparency support
- **Browser**: Uses Puppeteer (headless Chrome) for consistent rendering
- **Styling**: Modern gradient design with Apple-style typography

## File Naming Convention

- **HTML files**: Descriptive names (e.g., `speech-to-text.html`)
- **JavaScript files**: `capture-{feature-name}.js`
- **Generated screenshots**: Stored in `output/` directory as `{feature-name}.png` (e.g., `output/speech-to-text.png`, `output/speech-to-prompt.png`, `output/powered-by-gemini.png`, `output/powered-by-whisper.png`)
- **Manually provided assets**: Stored in `assets/` directory with descriptive names (e.g., `assets/dropdown.png`, `assets/settings.png`, `assets/settings-offline-models.png`) - these are not generated by scripts and must be updated manually

## Maintenance

When adding new screenshots:

1. Create HTML file in `html/` directory
2. Create capture script in `js/` directory
3. Update this README with new screenshot description
4. Test the capture script to ensure it works correctly
